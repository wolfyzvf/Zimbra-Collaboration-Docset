 27 /**
 28  * Creates a new autocomplete list. The list isn't populated or displayed until some
 29  * autocompletion happens. Takes a data class and loader, so that when data is needed (it's
 30  * loaded lazily), the loader can be called on the data class.
 31  * @class
 32  * This class implements autocomplete functionality. It has two main parts: matching data based
 33  * on keystroke events, and displaying/managing the list of matches. This class is theoretically
 34  * neutral concerning the data that gets matched (as long as its class has an <code>autocompleteMatch()</code>
 35  * method), and the field that it's being called from.
 36  * 
 37  * The data class's <code>autocompleteMatch()</code> method should returns a list of matches, where each match is
 38  * an object with the following properties:
 39  * <table border="1" width="50%">
 40  * <tr><td width="15%">data</td><td>the object being matched</td></tr>
 41  * <tr><td>text</td><td>the text to display for this object in the list</td></tr>
 42  * <tr><td>[key1]</td><td>a string that may be used to replace the typed text</td></tr>
 43  * <tr><td>[keyN]</td><td>a string that may be used to replace the typed text</td></tr>
 44  * </table>
 45  * 
 46  * The calling client also specifies the key in the match result for the string that will be used
 47  * to replace the typed text (also called the "completion string"). For example, the completion 
 48  * string for matching contacts could be a full address, or just the email.
 49  * 
 50  * The client may provide additional key event handlers in the form of callbacks. If the callback
 51  * explicitly returns true or false, that's what the event handler will return.
 52  * 
 53  * A single autocomplete list view may handle several related input fields. With the "quick complete" feature, there
 54  * may be multiple outstanding autocomplete requests to the server. Each request is managed through a context which
 55  * has all the information needed to make the request and handle its results.
 56  * 
 57  * 
 58  * 
 59  * Using Autocomplete
 60  * 
 61  * Autocomplete kicks in after there is a pause in the typing (that pause has to be at least 300ms by default). Let's say that
 62  * you are entering addresses into the To: field while composing an email. You type a few characters and then pause:
 63  * 
 64  * 	dav
 65  * 
 66  * ZCS will ask the user for people whose name or email address matches "dav", and display the matches in a list that pops up.
 67  * The matches will be sorted with the people you email the most at the top. When you select a match, that person's address
 68  * will replace the search string ("dav") in the To: field. Typically the address will be in a bubble.
 69  * 
 70  * 	Davey Jones x
 71  * 
 72  * Quick Complete
 73  * 
 74  * Many times you will know which address you're looking for, and you will type enough characters so that they will appear at
 75  * the top of the matches, and then you type semicolon or a return to select them once the list has come up. If you know that
 76  * the address you want will appear at the top of the matches based on what you've typed, then there's a way to select it 
 77  * without waiting for the list to come up: just type a semicolon. For example, let's assume that I email Davey Jones a lot,
 78  * and I know that if I type "dav" he will be the first match. I can just type
 79  * 
 80  * 	dav;
 81  * 
 82  * and continue, whether that's adding more addresses, or moving on to the subject and body (done easily via the Tab key).
 83  * Autocompletion will happen in the background, and will automatically replace "dav;" with the first match from the list. If 
 84  * no matches are found, nothing changes. One way to think of the Quick Complete feature is as the autocomplete version of 
 85  * Google's "I'm Feeling Lucky", though in this case you have a much better idea of what the results are going to be. You 
 86  * don't have to wait for the list to appear in order to add the bubble. It gets added for you.
 87  * 
 88  * You can type in multiple Quick Complete strings, and they will all be handled. For example, I could type
 89  * 
 90  * 	dav;pb;ann;x;
 91  * 
 92  * and see bubbles pop up for Davey Jones, Phil Bates, Ann Miller, and Xavier Gold without any more action on my part. I could
 93  * even type "dav;" into the To: field, hit Tab to go to the Cc: field, type "pb;" there, and then Tab to the Subject: field,
 94  * and start writing my message.
 95  * 
 96  * One small limitation of Quick Complete is that the bubbles will pop up within a field in the order that the results come 
 97  * back, which may not match the order of the strings you typed in. You can drag the bubbles to rearrange them if you want.
 98  * 
 99  * Special Keys
100  * 
101  * There are a number of keys that have special meanings when you are working with an input field that supports autocomplete. 
102  * Most of them apply while the list of matches is showing, and are used to control selection of the match you want:
103  * 
104  * Return		Adds the selected address
105  * Tab		Adds the selected address
106  * ;		Adds the selected address
107  * ,		Adds the selected address (if enabled in Preferences/Address Book/Autocomplete)
108  * DownArrow	Selects the next address (hold to repeat)
109  * UpArrow		Selects the previous address (hold to repeat)
110  * Esc		Hides the list
111  * 
112  * A few keys have special meanings while the list is not showing:
113  * 
114  * Return		If the input contains an email address, turn it into a bubble
115  * Tab		Go to the next field
116  * Esc		If requests are pending (it will say "Autocompleting"), cancel them. If not, cancel compose.
117  * 
118  * 
119  * 
120  * @author Conrad Damon
121  *
122  * @param {Hash}	params			a hash of parameters:
123  * @param	{String}		matchValue			the name of field in match result to use for completion
124  * @param	{function}		dataClass			the class that has the data loader
125  * @param	{function}		dataLoader			a method of dataClass that returns data to match against
126  * @param	{DwtComposite}	parent				the control that created this list (defaults to shell)
127  * @param	{String}		className			the CSS class
128  * @param	{Array}			delims				the list of delimiters (which separate tokens such as addresses)
129  * @param	{Array}			delimCodes			the list of delimiter key codes
130  * @param	{String}		separator			the separator (gets added to the end of a match)
131  * @param	{AjxCallback}	compCallback		the callback into client to notify it that completion happened
132  * @param	{AjxCallback}	selectionCallback	the callback into client to notify it that selection from extended DL happened (passed from email.js, and accessed from ZmDLAutocompleteListView.prototype._doUpdate)
133  * @param	{AjxCallback}	keyDownCallback		the additional client ONKEYDOWN handler
134  * @param	{AjxCallback}	keyPressCallback	the additional client ONKEYPRESS handler
135  * @param	{AjxCallback}	keyUpCallback		the additional client ONKEYUP handler
136  * @param	{string}		contextId			ID from parent
137  * @param	{Hash}			options				the additional options for the data class
138  * @param	{function}		locationCallback	used to customize list location (optional)
139  * 
140  * @extends		DwtComposite
141  */

248 /**
249  * Handles the on key down event.
250  * 
251  * @param	{Event}	event		the event
252  */
253 ZmAutocompleteListView.onKeyDown =
254 function(ev) {
255 
256 	ev = DwtUiEvent.getEvent(ev);
257 	var key = DwtKeyEvent.getCharCode(ev);
258 	var result = true;
259 	var element = DwtUiEvent.getTargetWithProp(ev, "_aclvId");
260 	DBG.println("ac", ev.type.toUpperCase() + " in " + element.id + ": " + key);
261 	var aclv = element && DwtControl.ALL_BY_ID[element._aclvId];
262 	if (aclv) {
263 		// if the user types a single delimiting character with the list showing, do completion
264 		var isDelim = (!ev.shiftKey && aclv._isDelimCode[key]);
265 		var visible = aclv.getVisible();
266 		aclv._actionHandled = false;
267 		// DBG.println("ac", "key = " + key + ", isDelim: " + isDelim);
268 		if (visible && aclv.handleAction(key, isDelim, element)) {
269 			aclv._actionHandled = true;
270 			result = false;
271 		}
272 
273 		aclv._inputValue[element.id] = element.value;
274 		var cbResult = aclv._runCallbacks(ZmAutocompleteListView.CB_KEYDOWN, element && element.id, [ev, aclv, result, element]);
275 		// DBG.println("ac", ev.type.toUpperCase() + " cbResult: " + cbResult);
276 		result = (cbResult === true || cbResult === false) ? cbResult : result;
277 	}
278 	if (AjxEnv.isFirefox){
279 		ZmAutocompleteListView.clearTimer();
280 		ZmAutocompleteListView.timer =  new AjxTimedAction(this, ZmAutocompleteListView.onKeyUp, [ev]);
281 		AjxTimedAction.scheduleAction(ZmAutocompleteListView.timer, 300)
282 	}
283 	return ZmAutocompleteListView._echoKey(result, ev);
284 };

337 /**
338  * "onkeyup" handler for performing autocompletion. The reason it's an "onkeyup" handler is that it's the only one
339  * that arrives after the input has been updated.
340  *
341  * @param ev		the key event
342  * 
343  * @private
344  */
345 ZmAutocompleteListView._onKeyUp =
346 function(ev) {
347 
348 	var element = DwtUiEvent.getTargetWithProp(ev, "_aclvId");
349 	if (!element) {
350 		return ZmAutocompleteListView._echoKey(true, ev);
351 	}
352 
353 	var aclv = DwtControl.ALL_BY_ID[element._aclvId];
354 	var key = DwtKeyEvent.getCharCode(ev);
355 	var value = element.value;
356 	var elId = element.id;
357 	DBG.println("ac", ev.type + " event, key = " + key + ", value = " + value);
358 	ev.inputChanged = (value != aclv._inputValue[elId]);
359 
360 	// reset timer on any address field key activity
361 	if (aclv._acActionId[elId] != -1 && !DwtKeyMap.IS_MODIFIER[key] && key != 9) {
362 		DBG.println("ac", "canceling autocomplete");
363 		AjxTimedAction.cancelAction(aclv._acActionId[elId]);
364 		aclv._acActionId[elId] = -1;
365 	}
366 
367 	// ignore modifier keys (including Shift), or a key with a modifier that makes it nonprintable
368 	if (DwtKeyMap.IS_MODIFIER[key] || DwtKeyMapMgr.hasModifier(ev)) {
369 		return true;
370 	}
371 
372 	// if the input is empty, clear the list (if it's for this input)
373 	if (!value && aclv._currentContext && element == aclv._currentContext.element) {
374 		aclv.reset(element);
375 		return true;
376 	}
377 
378 	// a Return following an address turns it into a bubble
379 	if ((key == 13 || key == 3) && aclv._complete(element)) {
380 		return false;
381 	}
382 
383 	// skip if input value is not changed
384 	if (!ev.inputChanged) {
385 		return true;
386 	}
387 
388 	ZmAutocompleteListView.clearTimer();
389 
390 	// regular input, schedule autocomplete
391 	var ev1 = new DwtKeyEvent();
392 	DwtKeyEvent.copy(ev1, ev);
393 	ev1.aclv = aclv;
394 	ev1.element = element;
395 	DBG.println("ac", "scheduling autocomplete for: " + elId);
396 
397 	var aif = DwtControl.ALL_BY_ID[element._aifId];
398 	if (aif && aif._editMode) {
399 		return false;
400 	}
401 	
402 	var acAction = new AjxTimedAction(aclv, aclv._autocompleteAction, [ev1]);
403 	aclv._acActionId[elId] = AjxTimedAction.scheduleAction(acAction, aclv._acInterval);
404 	
405 	return true;
406 };

493 // Kicks off an autocomplete cycle, which scans the content of the given input and then
494 // handles the strings it finds, possible making requests to the data provider.
495 ZmAutocompleteListView.prototype.autocomplete =
496 function(element) {
497 
498 	if (this._dataLoader && !this._dataLoaded) {
499 		this._data = this._dataLoader.call(this._dataClass);
500 		this._dataAPI = this._data;
501 		this._dataLoaded = true;
502 	}
503 
504 	var results = this._parseInput(element);
505 	this._process(results, element);
506 };

508 /**
509  * See if the text in the input is an address. If it is, complete it.
510  * 
511  * @param {Element}		element
512  * @return {boolean}	true if the value in the input was completed
513  */
514 ZmAutocompleteListView.prototype._complete =
515 function(element) {
516 
517 	var value = element.value;
518 	if (this._dataAPI.isComplete && this._dataAPI.isComplete(value)) {
519 		DBG.println("ac", "got a Return or Tab, found an addr: " + value);
520 		var result = this._parseInput(element)[0];
521 		var context = {
522 			element:	element,
523 			str:		result.str,
524 			isAddress:	true,
525 			isComplete:	result.isComplete,
526 			key:		this._getKey(result)
527 		}
528 		this._update(context);
529 		this.reset(element);
530 		return true;
531 	}
532 	return false;
533 };

535 // Parses the content of the given input by splitting the text at delimiters. Returns a list of
536 // objects with information about each string it found.
537 ZmAutocompleteListView.prototype._parseInput =
538 function(element) {
539 
540 	DBG.println("ac", "parse input for element: " + element.id); 
541 	var results = [];
542 	var text = element && element.value;
543 	if (!text) {
544 		return results;
545 	}
546 	DBG.println("ac", "PARSE: " + text);
547 	var str = "";
548 	for (var i = 0; i < text.length; i++) {
549 		var c = text.charAt(i);
550 		if (c == ' ' && !str) { continue; }	// ignore leading space
551 		var isDelim = this._isDelim[c];
552 		if (isDelim || c == ' ') {
553 			// space counts as delim if bubbles are on and the space follows an address
554 			var str1 = (this._dataAPI.isComplete && this._dataAPI.isComplete(str, true));
555 			if (str1) {
556 				DBG.println("ac", "parse input found address: " + str);
557 				str1 = (str1 === true) ? str : str1;
558 				results.push({element:element, str:str1, isComplete:true, isAddress:true});
559 				str = "";
560 			}
561 			else if (c == ";") {
562 				// semicolon triggers Quick Complete
563 				results.push({element:element, str:str, isComplete:true});
564 				str = "";
565 			}
566 			else {
567 				// space typed, but not after an address so no special meaning
568 				str += c;
569 			}
570 		}
571 		else {
572 			str += c;
573 		}
574 	}
575 	if (str) {
576 		results.push({str:str, isComplete:false});
577 	}
578 
579 	return results;
580 };

582 /**
583  * Look through the parsed contents of the input and make any needed autocomplete requests. If there is a 
584  * delimited email address, go ahead and handle it now. Also, make sure to cancel any requests that no
585  * longer match the contents of the input. This function will run only after a pause in the user's typing
586  * (via a setTimeout call), so existing contexts will be in either the REQUEST state or the DONE state.
587  */
588 ZmAutocompleteListView.prototype._process =
589 function(results, element) {
590 
591 	// for convenience, create a hash of current keys for this input
592 	var resultsHash = {};
593 	for (var i = 0; i < results.length; i++) {
594 		var key = this._getKey(results[i]);
595 		resultsHash[key] = true;
596 	}
597 	
598 	// cancel any outstanding requests for strings that are no longer in the input
599 	var pendingContextHash = {};
600 	var oldContexts = this._contexts[element.id];
601 	if (oldContexts && oldContexts.length) {
602 		for (var i = 0; i < oldContexts.length; i++) {
603 			var context = oldContexts[i];
604 			var key = context.key;
605 			if (key && context.reqId && context.state == ZmAutocompleteListView.STATE_REQUEST && !resultsHash[key]) {
606 				DBG.println("ac", "request for '" + context.str + "' no longer current, canceling req " + context.reqId);
607 				appCtxt.getAppController().cancelRequest(context.reqId);
608 				context.state = ZmAutocompleteListView.STATE_DONE;
609 				if (context.str == this._waitingStr) {
610 					this.setWaiting(false);
611 				}
612 			}
613 			else if (context.state == ZmAutocompleteListView.STATE_REQUEST) {
614 				pendingContextHash[context.key] = context;
615 			}
616 		}
617 	}
618 	
619 	// process the parsed content
620 	var newContexts = [];
621 	for (var i = 0; i < results.length; i++) {
622 		var result = results[i];
623 		var str = result.str;
624 		var key = this._getKey(result);
625 		var pendingContext = pendingContextHash[key];
626 		// see if we already have a pending request for this result; if so, leave it alone
627 		if (pendingContext) {
628 			DBG.println("ac", "PROCESS: propagate pending context for '" + str + "'");
629 			newContexts.push(pendingContext);
630 		}
631 		else {
632 			// add a new context
633 			DBG.println("ac", "PROCESS: add new context for '" + str + "', isComplete: " + result.isComplete);
634 			var context = {
635 				element:	element,
636 				str:		str,
637 				isComplete:	result.isComplete,
638 				key:		key,
639 				isAddress:	result.isAddress,
640 				state:		ZmAutocompleteListView.STATE_NEW
641 			}
642 			newContexts.push(context);
643 			if (result.isAddress) {
644 				// handle a completed email address now
645 				this._update(context);
646 			}
647 			else {
648 				// go get autocomplete results from the data provider
649 				this._autocomplete(context);
650 			}
651 		}
652 	}
653 	this._contexts[element.id] = newContexts;
654 };

688 /**
689  * Checks the given key to see if it's used to control the autocomplete list in some way.
690  * If it does, the action is taken and the key won't be echoed into the input area.
691  *
692  * The following keys are action keys:
693  *	38 40		up/down arrows (list selection)
694  *	37 39		left/right arrows (dl expansion)
695  *	27			escape (hide list)
696  *
697  * The following keys are delimiters (trigger completion when list is up):
698  *	3 13		return
699  *	9			tab
700  *	59 186		semicolon
701  *	188			comma (depends on user pref)
702  *
703  * @param {int}		key			a numeric key code
704  * @param {boolean}	isDelim		true if a single delimiter key was typed
705  * @param {Element}	element		element key event happened in 
706  * 
707  * @private
708  */
709 ZmAutocompleteListView.prototype.handleAction =
710 function(key, isDelim, element) {
711 
712 	DBG.println("ac", "autocomplete handleAction for key " + key + " / " + isDelim);
713 
714 	if (isDelim) {
715 		this._update();
716 	} else if (key == 37 && this._curExpanded) {
717 		// handle left key with an expanded DL
718 		this._memberListView._popdown();
719 
720 	} else if (key == 39) {
721 		// right arrow
722 		var dwttext = this._expandText && this._expandText[this._selected];
723 
724 		// if the caret is at the end of the input, expand a distribution list,
725 		// if possible
726 		if(!dwttext || Dwt.getSelectionStart(element) !== element.value.length) {
727 			return false;
728 		}
729 
730 		// fake a click
731 		dwttext.notifyListeners(DwtEvent.ONMOUSEDOWN);
732 
733 	} else if (key == 38 || key == 40) {
734 		// handle up and down arrow keys
735 		if (this._curExpanded) {
736 			return this._memberListView.handleAction(key, isDelim, element);
737 		}
738 		if (this.size() < 1) {
739 			return;
740 		}
741 		if (key == 40) {
742 			this._setSelected(ZmAutocompleteListView.NEXT);
743 		} else if (key == 38) {
744 			this._setSelected(ZmAutocompleteListView.PREV);
745 		}
746 	} else if (key == 27) {
747 		if (this.getVisible()) {
748 			this.reset(element); // ESC hides the list
749 		}
750 		else if (!this._cancelPendingRequests(element)) {
751 			return false;
752 		}
753 	} else if (key == 9) {
754 		this._popdown();
755 		return false;
756 	} else {
757 		return false;
758 	}
759 	return true;
760 };

937 // Updates the content of the input with the given match and adds a bubble
938 ZmAutocompleteListView.prototype._update =
939 function(context, match) {
940 
941 	context = context || this._currentContext;
942 	if (!context) {
943 		return;
944 	}

958 // continuation of _update
959 ZmAutocompleteListView.prototype._updateContinuation = 
960 function(context, match) {
961 
962 	var newText = "";
963 	var address = context.address = context.address || (context.isAddress && context.str) || (match && this._getCompletionValue(match));
964 	DBG.println("ac", "UPDATE: result for '" + context.str + "' is " + AjxStringUtil.htmlEncode(address));
965 
966 	var bubbleAdded = this._addBubble(context, match, context.isComplete);
967 	if (!bubbleAdded) {
968 		newText = address + this._separator;
969 	}
970 
971 	// figure out what the content of the input should now be
972 	var el = context.element;
973 	if (el) {
974 		// context.add means don't change the content (used by DL selection)
975 		if (!context.add) {
976 			// Parse the input again so we know what to replace. There is a race condition here, since the user
977 			// may have altered the content during the request. In that case, the altered content will not match
978 			// and get replaced, which is fine. Reparsing the input seems like a better option than trying to use
979 			// regexes.
980 			var results = this._parseInput(el);
981 			var newValue = "";
982 			for (var i = 0; i < results.length; i++) {
983 				var result = results[i];
984 				var key = this._getKey(result);
985 				if (context.key === key) {
986 					newValue += newText;
987 				}
988 				else {
989 					newValue += key;
990 				}
991 			}
992 			if (bubbleAdded) {
993 				newValue = AjxStringUtil.trim(newValue);
994 			}
995 			if (el.value !== newValue) {
996 				el.value = newValue;
997 			}
998 		}
999 		
1000 		if (!context.isComplete) {
1001 			// match was selected from visible list, refocus the input and clear the list
1002 			el.focus();
1003 			this.reset(el);
1004 		}
1005 	}
1006 	context.state = ZmAutocompleteListView.STATE_DONE;
1007 
1008 	this._runCallbacks(ZmAutocompleteListView.CB_COMPLETION, el && el.id, [address, el, match]);
1009 };

1011 // Adds a bubble. If we are adding it via Quick Complete, we don't want the input field to set
1012 // focus since the user may have tabbed into another input field.

1024 		}

1661 function(type, inputId, args) {
1662 
1663 	var result = null;
1664 	var list = this._callbacks[type];
1665 	if (list && list.length) {
1666 		for (var i = 0; i < list.length; i++) {
1667 			var cbObj = list[i];
1668 			if (inputId && cbObj.inputId && (inputId != cbObj.inputId)) { continue; }
1669 			var callback = cbObj.callback;
1670 			var r;
1671 			if (typeof(callback) == "function") {
1672 				r = callback.apply(callback, args);
1673 			}
1674 			else if (callback && callback.isAjxCallback) {
1675 				r = AjxCallback.prototype.run.apply(cbObj.callback, args);
1676 			}
1677 			if (r === true || r === false) {
1678 				result = (result == null) ? r : result && r;
